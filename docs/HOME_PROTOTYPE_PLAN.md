# План домашнего автоматического прототипа

## Цель

Проверить на ноутбуке и обычной USB-веб-камере полный минимальный цикл:

1. непрерывно получать видеопоток;
2. автоматически обнаруживать номерной знак;
3. автоматически распознавать его текст;
4. подтверждать номер несколькими кадрами;
5. применять правила допуска;
6. сохранять историю и фотографию без ручного нажатия кнопки.

Ручной снимок и регулярный ручной ввод номера в этот этап не входят.

## Технологии

- Python 3.12.10;
- FastALPR и ONNX Runtime на CPU;
- OpenCV для USB/IP-видеопотоков;
- PySide6 для интерфейса оператора;
- Argon2id для хеширования пароля администратора;
- SQLite как рабочее хранилище;
- DB Browser for SQLite для просмотра базы;
- openpyxl для экспорта Excel;
- pytest для автоматических тестов.

## Поток обработки

`камера → кадр → детектор номера → OCR → подтверждение по кадрам → правила → ручное/автоматическое решение → SQLite`

Камеры работают параллельно, но передают только свежие кадры в ограниченную
очередь. Распознавание выполняется централизованно: медленный ноутбук не накапливает
неограниченное количество старых кадров, а SQLite получает последовательные записи.

## Правила первой версии

- поддерживаются стандартные российские номера с двух- или трёхзначным регионом;
- ведущий ноль числовой части сохраняется;
- похожие латинские буквы преобразуются в допустимые кириллические;
- в чётный день допускается чётная первая цифра трёхзначной части, в нечётный —
  нечётная;
- при включённом ручном одобрении восьмичасовой интервал начинается только после
  кнопки `ЗАПРАВИЛСЯ`; кнопка `НЕ ЗАПРАВИЛСЯ` сохраняет результат без таймера;
- при выключенном ручном одобрении первое решение `РАЗРЕШЕНО` автоматически
  считается заправкой и запускает восьмичасовой интервал;
- низкая уверенность OCR не создаёт автоматического разрешения;
- стабильный, но некорректный по формату результат получает статус
  `ТРЕБУЕТСЯ ПРОВЕРКА`;
- повторное наблюдение того же номера в течение защитного интервала не создаёт
  дубликат, даже если номер увидела другая камера.

## Хранилище

SQLite хранит камеры, распознавания, решения, подтверждённые заправки и ошибки.
Фотографии сохраняются отдельными файлами, а база содержит путь к ним. Excel
используется только для отчётов, поэтому открытый Excel не блокирует работу камер.

Раздел администрирования защищён отдельным локальным паролем. При первой попытке
входа пароль создаётся, при следующих — проверяется. Очистка истории удаляет
события и фотографии, но сохраняет настройки, камеры, модели и хеш пароля.

## Границы этапа

В домашний прототип не входят пользователи и роли, управление шлагбаумом/колонкой,
защита от изменения истории администратором и
гарантированная промышленная точность. Эти задачи остаются в долгосрочной дорожной
карте `PROJECT_ROADMAP.md`.

## Критерии готовности

- программа стартует из PyCharm и открывает окно с живым видео;
- модели загружаются и выполняют локальный тестовый проход;
- SQLite создаётся автоматически;
- корректные и некорректные номера различаются;
- границы восьми часов проверяются тестами;
- ручной и автоматический запуск таймера проверяются отдельно;
- номер подтверждается несколькими кадрами;
- дубли между камерами подавляются;
- фотографии и события сохраняются автоматически;
- отчёт `.xlsx` создаётся и открывается в Excel;
- пароль администратора не хранится открытым текстом;
- очистка истории и фотографий проверяется на временной базе;
- отключение камеры не завершает программу, а запускает переподключение.
