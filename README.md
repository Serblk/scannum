# Домашний прототип фиксации автомобильных номеров

Программа показывает живое видео с USB-камеры, сама находит номерной знак,
распознаёт текст и подтверждает результат по нескольким кадрам. После этого она
применяет правила допуска, показывает решение оператору и сохраняет событие с
фотографией в локальную SQLite.

## Что уже работает

- одна или несколько USB/IP-камер из `config.toml`;
- локальное обнаружение номера и OCR через FastALPR/ONNX;
- нормализация похожих латинских и кириллических символов;
- стандартные российские номера с двух- и трёхзначным регионом;
- подтверждение одинакового результата по трём кадрам;
- защита от повторной фиксации одной машины разными камерами;
- правило чётности календарного дня;
- интервал восемь часов от подтверждённой фактической заправки;
- интерфейс с видео, рамками номеров, текущим решением и историей;
- переключаемый режим «Ручное одобрение»;
- защищённый раздел администрирования и согласованная очистка истории;
- SQLite, автоматические фотографии и экспорт `.xlsx`;
- журнал ошибок камеры и распознавания.

## Первый запуск в PyCharm

1. Откройте в PyCharm папку проекта целиком.
2. Убедитесь, что выбран интерпретатор `.venv` с Python 3.12.10.
3. Откройте `main.py` и нажмите зелёную кнопку запуска.
4. Откроется окно программы; для остановки закройте его.

Зависимости и модели на этом компьютере уже установлены. Для новой установки:

```powershell
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
.\.venv\Scripts\python.exe main.py check
```

При первом `check` потребуется интернет, чтобы один раз скачать модели в
`models/cache`. Дальнейшее распознавание выполняется локально.

## Настройка камер

Первая USB-камера задана в `config.toml` как `source = 0`. Для четырёх камер
добавьте ещё три блока с уникальными `id` и источниками `1`, `2`, `3`:

```toml
[[cameras]]
id = "camera-2"
name = "USB-камера 2"
source = 1
enabled = true
width = 1280
height = 720
```

Если индекс неизвестен, попробуйте значения начиная с нуля. Четыре камеры лучше
распределить по разным USB-портам или использовать качественный хаб с питанием.

## Команды

Запуск интерфейса с видео:

```powershell

```

Запуск прежнего консольного режима без окна:


Проверка моделей без камеры:

```powershell

```

Проверка подключения камер без запуска постоянного наблюдения:


```powershell

```

```powershell
.\.venv\Scripts\python.exe main.py confirm А030ВС77
```

## Ручное и автоматическое одобрение

Переключатель находится в верхней части окна и сохраняется в SQLite.

- **Ручное одобрение включено.** После `РАЗРЕШЕНО` оператор нажимает
  `ЗАПРАВИЛСЯ` или `НЕ ЗАПРАВИЛСЯ`. Восемь часов отсчитываются только после
  `ЗАПРАВИЛСЯ`. Необработанное событие остаётся в истории как ожидающее решения.
- **Ручное одобрение выключено.** Каждое автоматическое решение `РАЗРЕШЕНО`
  сразу считается состоявшейся заправкой и запускает восемь часов. Повторный
  приезд в течение интервала получает `ЗАПРЕЩЕНО`.

Переключение режима влияет на новые события. Старое ожидающее решение можно
выбрать в таблице и завершить кнопкой, даже если автоматический режим уже включён.

## Администрирование

Кнопка `Администрирование` находится в верхней части главного окна. При первом
входе программа предлагает создать пароль и повторить его. Минимальная длина —
10 символов. В SQLite сохраняется только Argon2id-хеш с солью, а не исходный
пароль.

При следующих входах требуется созданный пароль. После пяти неправильных попыток
доступ блокируется на 30 секунд. Авторизация действует только до закрытия окна
администрирования.

Команда `Очистить историю и фотографии` показывает количество удаляемых событий,
заправок, ошибок и файлов, затем требует подтверждение и повторный ввод пароля.
Удаляются:

- распознавания и решения;
- подтверждённые заправки;
- журнал ошибок;
- фотографии из `captures`.

Камеры, настройки, модели распознавания и пароль администратора сохраняются. Перед
транзакцией фотографии временно перемещаются; если SQLite не удалось очистить, они
возвращаются обратно. Встроенного восстановления забытого пароля в прототипе нет.

Экспорт истории в Excel:

```powershell
.\.venv\Scripts\python.exe main.py export
```

## Данные

- база: `data/system.db`;
- фотографии: `captures/<camera-id>/`;
- отчёты Excel: `reports/`;
- модели: `models/cache/`.

`system.db` можно открыть в DB Browser for SQLite. Во время работы программы базу
лучше только просматривать. Ручные изменения безопаснее выполнять после остановки
программы и создания резервной копии.

## Ограничения прототипа

Точность зависит от расстояния, угла, света, разрешения и выдержки веб-камеры.
Текущие модели являются универсальными, поэтому перед реальным использованием
потребуются испытания на дневных и ночных материалах. При четырёх камерах один CPU
может стать узким местом; тогда можно уменьшить частоту кадров, использовать
DirectML/CUDA или вынести распознавание в отдельные процессы.

Номера и фотографии являются чувствительными данными. Прототип не передаёт их в
интернет, но доступ к папке проекта должен быть ограничен владельцем компьютера.
# Версия для разработки и EXE

Исходный проект продолжает запускаться из PyCharm через `main.py` и использует
данные в папке проекта. Готовая сборка находится в `dist/ScanNum/ScanNum.exe`.
Она хранит базу, снимки, отчёты, настройки и модели отдельно:
`%LOCALAPPDATA%/ScanNum`.

Для повторной сборки:

```powershell
.\.venv\Scripts\python.exe -m pip install -r requirements-build.txt
powershell -ExecutionPolicy Bypass -File .\build_exe.ps1
```

`ExecutionPolicy Bypass` применяется только к этому одному запуску PowerShell и
не меняет системную политику. Для переноса на другой компьютер используется
`dist/ScanNum-0.4.0-win64.zip`: распакуйте весь архив и запустите `ScanNum.exe`.

## Выбор нескольких камер

После запуска камеры не подключаются автоматически. Нажмите **«Настройка камер»**,
дождитесь поиска встроенных и USB-источников, отметьте нужные камеры и нажмите
**«Подключить выбранные»**. Все выбранные потоки показываются сеткой и одновременно
передают кадры на распознавание. Выбор сохраняется в `camera_sources.json`, но при
следующем запуске оператор снова подтверждает подключение. Формат файла уже допускает
строковый источник для будущей RTSP-поддержки; добавление RTSP через интерфейс пока
недоступно.

## Время отображения результата

По умолчанию последний номер и решение показываются 10 секунд после того, как камера
перестала видеть этот номер. Период от 3 до 120 секунд задаётся в разделе
**«Администрирование»**. Пока тот же номер остаётся в кадре, отсчёт начинается заново.
Ожидающее ручное решение не очищается автоматически. При восьмичасовом запрете справа
также показывается точная дата и время следующей допустимой заправки.

## Стабилизация распознавания

Перед записью программа около трёх секунд собирает варианты российского номера из
одной области кадра. Требуется не менее четырёх наблюдений с уверенностью OCR от 70%.
Укороченный регион, например `С888КТ19`, поглощается устойчивым полным вариантом
`С888КТ197`. Если разные символы получили близкую поддержку, в SQLite создаётся одно
событие **«Требуется проверка»** со списком вариантов; автоматическая заправка и
восьмичасовой таймер в этом случае не запускаются. Невалидные форматы российских
автомобильных номеров до базы не доходят.
